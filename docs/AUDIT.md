# Аудит GO-GO v16.13 → v16.14 (2026-07-04)

Полная ревизия всех подсистем со сравнением с эталонными реализациями + тесты на
живой плате (serial-консоль отладочной сборки; сеть мака не трогалась).

владелец по чек-листу из README/SCENARIOS.

## 2. Радиофон окружения (Москва, квартира)

- **868 МГц (RU864, 864–870):** пол −110 dBm по всей полосе; единичные всплески
  −104 (865.1/865.9) и −95 (867.1) — датчики/сигнализации. Полоса практически
  чистая: порог автоперестройки −85 dBm даёт запас 25 dB, ложных hop'ов не будет.
- **2.4 ГГц:** 30+ сетей, каналы 1/6/11 заняты, сильнейшие соседи −54 dBm.
  Вывод: для шоу-контроля LoRa-канал в этом эфире на порядки надёжнее WiFi —
  архитектура выбрана верно.

## 4. Найдено и исправлено в v16.14

1. `Update.abort()` при оборванной/неудачной OTA (ручной и онлайн пути) — иначе
   OTA блокировалась до перезагрузки.
2. `startBLEMode` дёргал `WiFi.disconnect` до инициализации WiFi → ошибка
   `ESP_ERR_WIFI_NOT_INIT` в логе при каждой загрузке BLE/LoRa-режимов. Теперь
   с guard'ом.
3. Шум `nvs_get_str len fail` при первом старте — чтение кредов через `isKey`.
4. Debug-консоль ('q') оставляла WiFi-стек включённым в BLE-режиме — восстановление.

## 5. Известные ограничения (осознанные, в роадмапе)

- Панель в STA-режиме без пароля (локальная сеть площадки считается доверенной).
- OTA без подписи прошивки (проверка — только успешность записи).
- QLab-запрос ограничен одной UDP-датаграммой (большие воркспейсы → TCP OSC).
- `handleQlabQuery` блокирует loop до 900 мс — кнопка в этот момент не
  опрашивается (только пока открыта вкладка QLab).
- Джойн-флоу подтверждён логикой и логами; финальная валидация на телефоне — за
  владельцем (стенд без второго радио не может изобразить смартфон).

## 6. Инструментарий (остаётся в проекте)

Debug-сборка (`DEBUG_SERIAL 1` + FQBN `DebugLevel=info`) даёт serial-консоль:
`g` GO · `p` PANIC · `w` Web Setup · `W` WiFi onboarding · `s` статус ·
`e` свип LoRa-полосы · `q` скан 2.4 ГГц · `r` ребут. Релизная сборка
(`DEBUG_SERIAL 0`, `DebugLevel=none`) консоль не содержит.
