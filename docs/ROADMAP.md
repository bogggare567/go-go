# GO-GO — Roadmap и идеи

Рабочий документ. Здесь собраны все идеи по развитию прошивки, чтобы не повторять их заново в каждой сессии. Статус текущей прошивки: **v15, бета — всё работает, но UX «топорный», требует доводки**.

---

## 1. Радио: инициализация антенны и частотный план

**Сейчас:** частота захардкожена на EU 868 МГц, меню циклит только 868.0–869.0. Регион не выбирается, диапазон чипа не используется.

**Факты по железу:**
- Чип SX1262 умеет **150–960 МГц** программно.
- Реальное ограничение — согласование антенны и RF-тракта платы: Heltec V3 продаётся в вариантах **863–928 МГц** (EU/US/RU) и **470–510 МГц** (Китай). Штатная антенна-пружинка узкополосная.
- Юридические рамки по регионам (ISM): EU868 (863–870, duty cycle ≤1%), US915 (902–928), **RU864 (864–870, есть ограничения по полосам)**, AS923, AU915, IN865, KR920, CN470.

**План (по шагам):**
1. **Этап 1 — выбор региона.** Таблица частотных планов (регион → список каналов + макс. мощность). Выбор страны в меню при первом запуске, сохранение в Preferences. TX и RX должны совпадать по региону.
2. **Этап 2 — точная подстройка.** Ручной ввод частоты с шагом 100 кГц / 25 кГц прямо на устройстве (меню) и/или через WiFi-интерфейс.
3. **Этап 3 — автоподбор чистой полосы.** Устройство сканирует каналы региона и меряет фоновый RSSI (spectral scan), выбирает самый тихий канал. В RadioLib для SX1262 есть `scanChannel()` (CAD — Channel Activity Detection) и режим RX для замера RSSI.
4. **Этап 4 — master/slave согласование.** Гейтвей (RX) = master: выбирает канал, объявляет его на служебном канале (фиксированная «домашняя» частота региона). Пульты (TX) = slave: слушают служебный канал, получают рабочий канал, перепрыгивают. При потере связи — возврат на служебный канал. Это же решает вопрос «несколько пар в одном здании».

**Референс:** Meshtastic (github.com/meshtastic/firmware) — готовые таблицы частотных планов по странам и логика слотов, можно взять структуру их `RegionInfo`.

## 2. BLE: переназначаемые клавиши

**Сейчас:** GO = Space, PANIC = Esc, зашито в HID-репорте.

**План:**
- По умолчанию оставить Space/Esc (QLab).
- Настройка «GO key» / «PANIC key» из списка популярных: Space, Enter, Esc, стрелки, F1–F12, PgUp/PgDn, буквы — покрывает QLab, Ableton, PowerPoint, MultiPlay, SCS.
- Хранить в Preferences, менять через меню на устройстве и через WiFi-интерфейс.
- Пресеты: «QLab», «Ableton», «PowerPoint / презентации» — хорошая фича для README.

---

## 3. WiFi-интерфейс настройки

**Сейчас:** только портал WiFiManager при первичной настройке (IP/порт/адреса OSC).

**План:** полноценная веб-страница настроек (ESPAsyncWebServer), доступная в рабочем режиме:
- регион/частота/мощность LoRa, автоскан;
- переназначение BLE-клавиш;
- OSC-адреса и цели;
- статус линка (RSSI/SNR, счётчики пакетов) в реальном времени;
- OTA-обновление прошивки (ArduinoOTA / ElegantOTA) — важно, когда устройств несколько.

---

## 4. Безопасность LoRa-канала

Пакеты сейчас без аутентификации — любой на канале может послать GO/PANIC. Для театра критично.
- Добавить shared key (задаётся при паринге) + HMAC/CMAC подпись пакета, счётчик уже есть (anti-replay частично реализован через `counter`).
- Продавать в README как фичу: «защищённый канал».

## 5. Рефакторинг: разбить монолит

`LoRa_soundkorb.ino` — 2790 строк. Целевая структура (Arduino IDE собирает все файлы в папке скетча):

```
LoRa_soundkorb.ino   — setup/loop, склейка модулей
config.h             — пины, константы, структуры конфигов
settings.cpp/.h      — Preferences load/save
radio.cpp/.h         — LoRa: init, пакеты, ACK, beacon, частотные планы
ble_hid.cpp/.h       — BLE HID + keymap
osc_wifi.cpp/.h      — WiFi, WiFiManager, OSC
ui_screens.cpp/.h    — все draw*-функции, экраны
ui_input.cpp/.h      — кнопка, клики/удержания, меню
battery.cpp/.h       — заряд, LED
```

Правило: сначала разбиение 1:1 без изменения логики (проверить, что бинарь собирается и ведёт себя так же), потом уже новые фичи.
