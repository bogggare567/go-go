# GO-GO — Roadmap и идеи

Рабочий документ. Здесь собраны все идеи по развитию прошивки, чтобы не повторять их заново в каждой сессии. Статус текущей прошивки: **v15, бета — всё работает, но UX «топорный», требует доводки**.

---

## 1. Радио: инициализация антенны и частотный план

**Сейчас:** частота захардкожена на EU 868 МГц, меню циклит только 868.0–869.0. Регион не выбирается, диапазон чипа не используется.

**Факты по железу:**
- Чип SX1262 умеет **150–960 МГц** программно.
- Реальное ограничение — согласование антенны и RF-тракта платы: Heltec V3 продаётся в вариантах **863–928 МГц** (EU/US/RU) и **470–510 МГц** (Китай). Штатная антенна-пружинка узкополосная.
- Юридические рамки по регионам (ISM): EU868 (863–870, duty cycle ≤1%), US915 (902–928), **RU864 (864–870, есть ограничения по полосам)**, AS923, AU915, IN865, KR920, CN470.

**План (по шагам):**
1. ✅ **Этап 1 — выбор региона** (v16.2, меню Region; EU868 хранит старый список каналов для совместимости). Таблица частотных планов (регион → список каналов + макс. мощность). Выбор страны в меню при первом запуске, сохранение в Preferences. TX и RX должны совпадать по региону.
2. ~~Этап 2 — точная подстройка~~ — сделана в v16.2, **убрана в v16.4 по решению владельца** (не нужна). Мощность TX теперь всегда = максимум региона (тоже решение владельца, v16.4).
3. ✅ **Этап 3 — автоподбор чистой полосы** (v16.5): гейтвей на старте сканирует сетку 0.2 МГц по всей полосе региона и встаёт на самый тихий бин. Устройство сканирует каналы региона и меряет фоновый RSSI (spectral scan), выбирает самый тихий канал. В RadioLib для SX1262 есть `scanChannel()` (CAD — Channel Activity Detection) и режим RX для замера RSSI.
4. ✅ **Этап 4 — master/slave** (v16.5, дизайн согласован с владельцем 2026-07-02):
   - Ведущий = RX (стационарный, у компа), ведомый = TX.
   - «Свой/чужой» — цифровой беакон (ID+CRC), а не морзянка по эфиру; **морзянка GO — на светодиоде** ведомого при обнаружении своего ведущего.
   - Поиск: TX CAD-сканирует сетку (2 мс/бин), на активных бинах ждёт беакон до 2.6 с, сверяет ID пары.
   - Переход: RX шлёт 5× PKT_HOP (индекс нового бина) на старом канале → оба прыгают; прозевавший TX просто ищет заново.
   - Гистерезис: hop только если фон >-90 dBm стабильно 6 с, нет GO/PANIC 10 с, и ≥60 с после прошлого hop.
   - **Ручной режим**: Freq: Auto → 868.0 → … → Auto; фиксированный канал полностью выключает автоматику.
   - Служебный канал не нужен (поиск заменяет его).

**Референс:** Meshtastic (github.com/meshtastic/firmware) — готовые таблицы частотных планов по странам и логика слотов, можно взять структуру их `RegionInfo`.

## 2. BLE: переназначаемые клавиши

**Сейчас:** GO = Space, PANIC = Esc, зашито в HID-репорте.

**Нюансы текущего поведения (от владельца, 2026-07-02):**
- Работает нормально, но **нет автоподключения**: если хост уже был сопряжён, после включения устройства коннект надо инициировать с компьютера. Нужен BLE bonding (сохранение ключей) + directed advertising к известному хосту → подключение само.
- Это эмуляция клавиатуры → нажатия идут в активное окно; в доке предупреждать «держите QLab в фокусе» (в README и SCENARIOS уже добавлено).

**План:**
- ✅ Автоподключение (v16.3): включён bonding — macOS сам переподключается, когда видит advertising. Проверить на живом маке; если не хватит — добавить directed advertising.
- По умолчанию оставить Space/Esc (QLab).
- ✅ Настройка «GO key» / «PANIC key» (v16.3, меню): Space, Enter, Esc, стрелки, PgUp/PgDn, F5, B. Расширение списка/пресеты — при WiFi-интерфейсе.
- Хранить в Preferences, менять через меню на устройстве и через WiFi-интерфейс.
- Пресеты: «QLab», «Ableton», «PowerPoint / презентации» — хорошая фича для README.

## 3. WiFi-интерфейс настройки

✅ **Сделано в v16.6** (web_ui.cpp + web_html.h, встроенный WebServer без внешних библиотек):
- меню Web Setup → AP `GO-GO-XXXX`/`192.168.4.1` в любом режиме; в OSC-режимах панель всегда на IP платы;
- статус (линк, RSSI/SNR, батарея, счётчики), кнопки GO/PANIC, режим/выход RX, регион, частота Auto/фикс, OSC-цель, BLE-клавиши;
- живая спектрограмма (зеркалится с OLED, пауза линка — задокументирована);
- OTA: загрузка .bin прямо со страницы (/update, Update.h).

**v16.7:** WiFiManager выброшен — первичная настройка WiFi тоже через нашу панель (SSID/пароль на странице, креды в NVS); ручная частота = вся сетка 0.2 МГц (35 каналов), не 5; BLE ставится на паузу пока открыт Web Setup (один радиотракт 2.4 ГГц); онлайн-обновление: панель проверяет firmware/version.json в репо и ставит прошивку по HTTPS без компьютера.

**v16.8:** captive portal — панель всплывает сама после подключения к сети платы (DNS-ловушка + 302 на пробы iOS/Android); вкладка **QLab** — плата как мост OSC↔HTTP: транспорт (GO/Panic/Pause/Resume/Stop/Prev/Next/Load/Reset), список кью (/cueLists → плейхед/старт по номеру), активные кью (/runningOrPausedCues, поллинг 2.5 с), /connect с паролем. Рендер весь в браузере, плата только пересылает JSON.

**v16.9 (по фидбеку владельца):** WiFi Setup отделён от Web Setup. WiFi Setup = страница-онбординг на AP платы (скан сетей с уровнем сигнала, тап по сети, пароль с «глазком», статус подключения, IP в новой сети, авторебут); полная панель — в локальной сети по IP платы. Починен «призрак»: WiFi.persistent(false) + убран фолбэк на сохранённую в ESP сеть — после factory reset плата ни к чему сама не подключается.

**Осталось на потом:** пароль/авторизация панели в STA-режиме (сейчас открыта в локальной сети площадки), mDNS-имя (gogo.local); **OSC по TCP** для больших проектов QLab (UDP-ответ /cueLists ограничен ~1.4 КБ — на больших воркспейсах список обрежется); редактирование уровней/гео из панели (мост уже позволяет — нужен UI).

## 5а. Полировка UX (недочёты из аудита кнопок 2026-07-02)

Аудит показал: расхождений «написано ↔ делает» нет, но на четырёх экранах есть
**ненаписанные** действия — работают, но пользователь о них не знает:

- [ ] `MODE INFO` — клик возвращает в меню, hold на GO-экран; подсказки нет вовсе
- [ ] `SEARCH RX / WAIT TX` — написано только «hold: menu», а клик делает retry
- [ ] `NO BLE` (и NO_CONNECTION у RX-гейтвея) — клик перезапускает выход, не написано
- [ ] `STATUS` — подсказка убрана намеренно (не перегружать), но клик→меню, hold→GO;
      решить: добавить строку или оставить
- [ ] Требование к новым экранам: каждое активное действие кнопки должно быть написано на экране
- [x] v16.4: hold-действия срабатывают во время удержания (не после отпускания) с прогресс-баром;
      экран подтверждения смены режима убран («не успеваешь согласиться» — жалоба владельца)

## 5б. Duty cycle EU868 (заметка на будущее)

Пинг каждые 2 с + беакон каждые 2 с ≈ 1.5% занятости канала — формально выше лимита
1% для 868.0–868.6. Варианты: увеличить интервалы до 3 с, или беакон только при
отсутствии трафика, или канал 869.525 (10% duty). Решить при этапе автоскана.
